# Set Up Pillar SDK for My App

Analyze my codebase and create a complete Pillar SDK integration with tools, context sync, and handlers.

## What Pillar Does

Pillar provides:
- A co-pilot chat panel that answers user questions using your knowledge base
- Tools that the AI can invoke directly in your app
- Plans that guide users through multi-step workflows

### 3. Generate Tools (10-30 based on my app)
Look at my routes, features, modals, and APIs to generate tools using `usePillarTool`:

```tsx
// hooks/usePillarTools.ts
import { usePillarTool } from '@pillar-ai/react';
import { useRouter } from 'next/navigation';

export function usePillarTools() {
  const router = useRouter();

  // Navigation tools based on my routes
  usePillarTool({
    name: 'open_settings',
    type: 'navigate',
    description: 'Navigate to the settings page',
    examples: ['go to settings', 'open settings'],
    autoRun: true,
    execute: () => { router.push('/settings'); },
  });

  // Trigger tools for modals/wizards
  usePillarTool({
    name: 'invite_member',
    type: 'trigger_tool',
    description: 'Invite a new team member',
    inputSchema: {
      type: 'object',
      properties: {
        email: { type: 'string', description: 'Email address to invite' },
        role: { type: 'string', description: 'Role: admin, member, or viewer' },
      },
      required: ['email'],
    },
    execute: ({ email, role }) => { openInviteModal({ email, role }); },
  });

  // Query tools that fetch data for the AI
  usePillarTool({
    name: 'list_projects',
    type: 'query',
    description: 'List all projects the user has access to',
    autoRun: true,
    autoComplete: true,
    execute: async () => {
      const projects = await api.getProjects();
      return projects.map(p => ({ id: p.id, name: p.name }));
    },
  });

  // Admin-only tools with requiredContext
  usePillarTool({
    name: 'delete_user',
    type: 'trigger_tool',
    description: 'Delete a user from the organization',
    requiredContext: { userRole: 'admin' },
    inputSchema: {
      type: 'object',
      properties: {
        userId: { type: 'string', description: 'ID of the user to delete' },
      },
      required: ['userId'],
    },
    execute: async ({ userId }) => {
      await api.deleteUser(userId);
      return { deleted: true };
    },
  });
}
```

### 4. Create RouteContextSync Component
Sync the current route to Pillar so the AI knows where the user is:

```tsx
// components/RouteContextSync.tsx
'use client';
import { usePillar } from '@pillar-ai/react';
import { usePathname } from 'next/navigation';

export function RouteContextSync() {
  const { pillar } = usePillar();
  const pathname = usePathname();
  
  useEffect(() => {
    pillar?.setContext({ currentPage: pathname });
  }, [pillar, pathname]);
  
  return null;
}
```

### 5. Mount Tools Component
Include your tools hook in a mounted component:

```tsx
// components/PillarTools.tsx
'use client';
import { usePillarTools } from '@/hooks/usePillarTools';

export function PillarTools() {
  usePillarTools();
  return null;
}
```

```tsx
export const agentGuidance = `
PREFER API OVER NAVIGATION:

- API tools execute instantly and return structured results

VERIFY BEFORE COMMITTING:
- When creating resources, test the query or configuration first to confirm it works
- Only create after validation succeeds

```

Then sync with `npx pillar-sync --scan ./src`.
